# Security Policy

## Reporting a vulnerability

Please report suspected vulnerabilities through
[GitHub Private Vulnerability Reporting](https://github.com/2x11-xyz/euler/security/advisories/new).
Do not open a public issue, publish a proof of concept, or include credentials
or other sensitive data in the report.

Useful reports explain the affected version or commit, the impact, steps to
reproduce, and any suggested mitigation. The maintainer will assess the report
privately and coordinate disclosure with the reporter when a fix is needed.

## Scope

This policy covers the Euler source repository, its official GitHub Actions
workflows, and release artifacts published from this repository. Vulnerabilities
in third-party providers or dependencies should also be reported to their
respective maintainers.
